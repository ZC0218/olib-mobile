# 📋 Olib 项目更新内容

> 日期：2026-02-18  
> 版本：v1.1.0 开发阶段

---

## 一、后端架构重构

### 1.1 架构重新规划（02-17）

移除 Z-Library 代理功能，围绕三大核心职责重建后端：

| 模块 | 职责 | 状态 |
|------|------|------|
| 微信认证 | 服务号扫码授权，接入微信回调 | ✅ 已完成 |
| 用户管理 | 注册、权限控制、角色管理 | ✅ 已完成 |
| AI 转发 | DeepSeek 对接，智阅锦囊推荐 | ✅ 已完成 |

- 数据库：`olibserver`
- AI 引擎：DeepSeek
- 认证方式：微信服务号二维码扫码授权 → 控制 AI 功能访问，基础服务保持开放

### 1.2 API 接口对齐（02-18）

- 新增 `POST /api/prescribe` 接口，接入前端 AI 智阅锦囊功能
- 统一 API 响应格式为 `{ success: bool, data: ..., error: ... }`
- 后端 Schema 与前端数据模型严格对齐

### 1.3 用户标识迁移至 OpenID（02-18）

- 将主要用户标识从 `device_id` 迁移为微信 `openid`
- 支持多设备共享微信授权
- 新设备使用已有 OpenID 可直接授权， 已有设备自动更新 device_id
- 修改 `wx_callback_event` 和 `UserDAO` 相关逻辑

## 二、前端功能更新

### 2.1 AI 智阅锦囊集成（02-12 ~ 02-14）

根据 `2026-02-12-AI智阅锦囊更新计划.md` 完成前端侧开发：

- ✅ 新增 `prescriber_screen.dart` — AI 锦囊主页面
- ✅ 新增 `ai_service.dart` — AI 服务抽象层
- ✅ 新增 `backend_api.dart` — 后端 API 客户端（`/auth/*` 接口封装）
- ✅ 首页入口卡片 → AI 锦囊推荐 → 搜索/下载 一站式流程
- ✅ 修复推荐后返回白屏问题
- ✅ 修复搜索页面白屏问题

### 2.2 书籍详情页重设计（02-14）

- 毛玻璃（Glassmorphism）Hero 区域
- 改进排版和元数据展示
- 操作按钮样式优化
- 收藏图标状态正确反映已保存状态

### 2.3 收藏功能调整（02-14）

- 禁用 QR 码导入/导出功能（数据密度问题）
- 移除收藏页 QR 扫描入口
- 隐藏分享预览中的 QR 码生成

### 2.4 存储服务迁移（02-14）

- `StorageService` 从 `shared_preferences` 迁移至 **Hive**
- 更新 Provider 注入方式（Riverpod）
- 移除 `shared_preferences` 依赖

### 2.5 认证流程修复（02-18）

- 修复 QR 扫码授权流程中的竞态条件
- 改进错误处理逻辑
- 确保授权成功/失败后的页面跳转顺畅
- 适配后端 OpenID 为主标识的变更

## 三、技术改进

| 类别 | 变更 | 影响范围 |
|------|------|----------|
| 数据存储 | shared_preferences → Hive | 全局存储层 |
| 用户标识 | device_id → openid | 认证 & 用户管理 |
| API 格式 | 统一 `{success, data, error}` | 全部后端接口 |
| AI 服务 | Mock → DeepSeek 实际调用 | 智阅锦囊功能 |
